## 📌 Executive Summary

**製品名**: クルマログ  
**コンセプト**: 個人向けカーメンテナンス記録管理アプリ  
**対象ユーザー**: 車を所有し、メンテナンス履歴を整理したい個人  
**解決する問題**: 「いつガソリン給油したっけ？」「次のオイル交換はいつ？」という記憶の曖昧さ

---

## 🎯 ビジョン・ミッション

### ビジョン
車のメンテナンスを「習慣化」し、「予防的」に管理できるツール

### ミッション
- シンプルに記録できる（3タップで完了）
- 忘れずに通知する（次回予定を自動計算）
- データで見える化する（支出・走行距離推移）

---

## 👤 ターゲットユーザー・ペルソナ

### Primary ペルソナ：「めんどくさがり屋の車好き」
- **年齢**: 25-50歳
- **車への関心**: 中程度（大事だけど詳しくない）
- **現在の課題**:
  - メンテナンス履歴を手帳・メモで管理している
  - 次の予定を忘れてしまう
  - ガソリン代・修理費がいくらかかってるか不明
- **期待値**: 「手軽に記録して、自動で思い出してくれる」

## 📋 機能要件（MVP）

### レベル 1：コア機能（必須）

#### 1.1 メンテナンス項目の記録
**対象メンテナンス:**
- ⛽ **ガソリン給油** → 日付、給油量（L）、金額、走行距離
- 🛢️ **エンジンオイル交換** → 日付、走行距離、金額、メモ（使用オイル種類など）
- 🛞 **タイヤ交換** → 日付、走行距離、金額、メモ（タイヤサイズ、ブランドなど）
- 🚿 **洗車** → 日付、メモ（外装、内装など）
- ⚠️ **故障・修理** → 日付、内容、走行距離、金額、メモ

**入力方法:**
- Web フォーム（シンプル UI）
- 必須項目：日付、メンテナンス種類
- オプション項目：走行距離、金額、メモ

**UI フロー:**
```
[ホーム画面]
  ↓
[+ ボタン（新規記録）]
  ↓
[メンテナンス種類選択] → [ガソリン給油, オイル交換, ...]
  ↓
[入力フォーム] → [日付, 走行距離, 金額, メモ]
  ↓
[保存] → [ホーム画面に追加]
```

#### 1.2 記録一覧表示
- **メンテナンス履歴表**：全記録を時系列（新しい順）で表示
- **列**: 日付、種類、走行距離、金額、メモ
- **ソート**: 日付（昇順/降順）、種類でフィルタ
- **編集・削除**: 各レコードを編集・削除可能

#### 1.3 次回メンテナンス予定
**予定計算ロジック:**

| メンテナンス | 予定計算 | 基準 |
|-----------|--------|------|
| オイル交換 | 最後の交換から 5,000km または 6 ヶ月 | 一般的な乗用車 |
| タイヤ交換 | 走行距離 50,000km 時点で提案（消耗率による） | 推奨時期 |
| ガソリン給油 | 残量 25% 以下で通知（手動設定） | 燃費目安 |
| 洗車 | 推奨：月 1 回（カスタマイズ可） | 汚れ防止 |

**表示内容:**
```
【次回予定】
- エンジンオイル交換: あと 1,200km（2026-04-15 予想）
- タイヤ交換: あと 12,000km（2026-06-01 予想）
- 洗車: 前回から 30 日経過（推奨実施）
```

#### 1.4 ダッシュボード（統計・可視化）

**表示項目:**
- 📊 **月別支出グラフ** → ガソリン代、メンテナンス費を円グラフ・棒グラフで表示
- 🛣️ **走行距離推移** → グラフ（直近 12 ヶ月の走行距離/月）
- 📈 **平均燃費** → (総給油量) / (走行距離)
- 💰 **総支出** → 累計ガソリン代 + メンテナンス費
- 🔔 **アラート** → 「あと 500km でオイル交換」など

## 💾 データモデル

### エンティティ：`maintenance_records`

```javascript
{
  id: UUID,                           // 一意の ID
  type: "gasoline" | "oil" | "tire" | "wash" | "repair",  // メンテナンス種類
  date: Date,                         // 実施日
  mileage: Number,                    // 走行距離（km）- オプション
  amount: Number,                     // 金額（円） - オプション
  memo: String,                       // メモ（最大 500 文字） - オプション
  created_at: Timestamp,              // 記録作成日時
  updated_at: Timestamp               // 最終更新日時
}
```

### 設定テーブル：`user_settings`

```javascript
{
  userId: String,                     // ユーザー ID（MVP では固定）
  fuel_alert_threshold: Number,       // ガソリン残量アラート（%）デフォルト: 25
  oil_change_interval_km: Number,     // オイル交換推奨距離（km）デフォルト: 5000
  oil_change_interval_months: Number, // オイル交換推奨期間（月）デフォルト: 6
  car_name: String,                   // 車の名前/型式（メモ用）
  current_mileage: Number,            // 現在の走行距離（ダッシュボード用）
}
```

## 🔧 技術仕様

### スタック

| 層 | 技術 | 理由 |
|----|------|------|
| **フロント** | Vue.js 3 + Vite | 軽量・高速・学習コスト低 |
| **バック** | Node.js + Express | 既存環境と統一 |
| **DB** | SQLite（MVP） / PostgreSQL（本番） | シンプル・管理不要 |
| **ホスト** | VPS（既存） | セットアップ済み |

### API 設計（REST）

```
GET  /api/maintenance          → 全記録取得
POST /api/maintenance          → 新規記録作成
GET  /api/maintenance/:id      → 記録詳細取得
PUT  /api/maintenance/:id      → 記録更新
DELETE /api/maintenance/:id    → 記録削除

GET  /api/dashboard            → ダッシュボードデータ取得
GET  /api/dashboard/upcoming   → 次回予定取得
GET  /api/dashboard/stats      → 統計データ取得

GET  /api/settings             → 設定取得
PUT  /api/settings             → 設定更新
```
