# クルマログ × MCP（Model Context Protocol）アーキテクチャ

**概念**: Claude がクルマログのデータに直接アクセス → シームレスな AI 連携  
**PM**: クロー  
**日付**: 2026-03-01

---

## 🔗 MCP とは（Model Context Protocol）

### **Anthropic の MCP の役割**

```
従来:
ユーザー ← → Claude（テキストだけ）
           ↗ 外部データやツール（別途統合が必要）

MCP を使った新しい形:
ユーザー ← → Claude ← → [MCP を通じて接続]
                            ├─ クルマログのデータ
                            ├─ 外部 API
                            └─ その他ツール
```

**簡潔に言うと：**

```
「Claude が外部データソース・ツール・API に
 標準的なプロトコル（MCP）で接続できる仕組み」
```

### **理想的な使い方**

```
【ユーザーの体験】

ユーザー: 「Claude、最近の燃費トレンドを分析して」

Claude（MCP 経由）:

→ すべてシームレス。ユーザーはテキストで質問するだけ
```

## 📋 MCP の実装詳細

### **1. MCP サーバー（クルマログ側）**

```javascript
// mcp-server.js

const MCPServer = require('@anthropic-ai/sdk/mcp');

const server = new MCPServer({
  name: 'kuruma-log-mcp',
  version: '1.0.0'
});

// リソース定義：IndexDB のデータを公開
server.defineResource('maintenance/records', async () => {
  return await getMaintenanceRecords();
});

server.defineResource('maintenance/upcoming', async () => {
  return await getUpcomingMaintenance();
});

server.defineResource('stats/mileage', async () => {
  return await calculateMileageStats();
});

server.defineResource('stats/fuel-efficiency', async () => {
  return await calculateFuelEfficiency();
});

// ツール定義：Claude が実行可能な操作
server.defineTool('add_record', {
  description: 'メンテナンス記録を追加',
  parameters: {
    type: 'object',
    properties: {
      type: { type: 'string', enum: ['gasoline', 'oil', 'tire', 'wash', 'repair'] },
      date: { type: 'string' },
      mileage: { type: 'number' },
      amount: { type: 'number' },
      memo: { type: 'string' }
    },
    required: ['type', 'date']
  },
  handler: async (params) => {
    return await addMaintenanceRecord(params);
  }
});

server.defineTool('analyze_trend', {
  description: '燃費・メンテナンストレンドを分析',
  parameters: {
    type: 'object',
    properties: {
      metric: { type: 'string', enum: ['fuel_efficiency', 'maintenance_cost', 'mileage'] },
      period_months: { type: 'number', default: 12 }
    }
  },
  handler: async (params) => {
    return await analyzeTrend(params);
  }
});

server.start();
```

### **2. Claude 側の使い方（MCP クライアント）**

```
ユーザー: 「最近の燃費が悪い気がするんだけど」

Claude:
1. MCP 経由で "fuel_efficiency" リソースにアクセス
2. 過去データを取得
3. analyze_trend ツールを実行
4. 結果に基づいて回答

レスポンス:
「あなたの場合、過去 3 ヶ月で 8.5km/L → 7.8km/L に低下しています。
 
 考えられる原因:
 1. エンジンオイルが古い可能性（最後の交換から 4,800km）
 2. タイヤの空気圧が低い
 3. エアフィルタの交換時期
 
 クルマログに記録がある次回予定:
 - オイル交換: あと 200km
 
 修理を早めることをお勧めします。」
```

## 🚀 実装パターン 3 つ

### **パターン A: 「ローカル MCP サーバー」（推奨）**

```
【構成】
クルマログ側で MCP サーバーを起動
├─ localhost:5000 で MCP サーバー起動
├─ IndexDB を公開
└─ Claude がアクセス

【メリット】
✅ すべてローカル（プライバシー完全）
✅ インターネット不要
✅ 高速
✅ セキュリティ最高

【実装難度】
中程度（Node.js MCP ライブラリを使用）

【タイミング】
Phase 2（後日）
```

---

### **パターン B: 「クラウド MCP サーバー」**

```
【構成】
クルマログの IndexDB データを定期的に
クラウド（Firebase / Vercel）に同期
├─ クラウド側で MCP サーバー起動
├─ Claude がクラウド経由でアクセス
└─ リアルタイム同期

【メリット】
✅ 複数デバイス間で同期可能
✅ バックアップ機能
✅ 拡張性高い

【デメリット】
❌ プライバシーの懸念
❌ クラウドコスト

【タイミング】
Phase 3（さらに後）
```

デフォルト: ローカル MCP サーバー

【メリット】
