## 📊 デプロイ先の選択肢（4つ）

### **選択肢 1: Vercel（推奨）**

**特徴:**
```
- SPA（Vue.js）の最適デプロイ先
- ビルド・デプロイが自動（GitHub 連携）
- エッジ関数（サーバーレス）対応
- 無料枠: 月間 100GB の帯域幅
```

**プロセス:**
```
1. GitHub に リポジトリアップロード
2. Vercel に GitHub アカウントで接続
3. Deploy ボタン → 自動ビルド・デプロイ
4. kuruma-log.vercel.app で公開

以降、GitHub に push すると自動デプロイ
```

**メリット:**
- ✅ 最速デプロイ（約 1 分）
- ✅ 自動 HTTPS
- ✅ ダッシュボードで監視
- ✅ 無料プラン充実
- ✅ カスタムドメイン対応（有料）

**デメリット:**
- ❌ カスタムドメインは有料（¥800/月程度）
- ❌ Vercel に依存

**費用:**
```
無料プラン: ¥0
カスタムドメイン: ¥800-1,000/月
```

**適用場面:**
```
✅ 個人・小規模プロジェクト
✅ 無料で始めたい
✅ 迅速なデプロイが必要
```

---

### **選択肢 2: GitHub Pages（最安）**

**特徴:**
```
- GitHub のリポジトリから直接ホスティング
- 完全無料
- 静的サイト（SPA）に最適
- GitHub Actions で自動ビルド
```

**プロセス:**
```
1. GitHub リポジトリ作成
2. GitHub Pages を有効化（Settings）
3. npm run build
4. dist/ を gh-pages ブランチにデプロイ
5. username.github.io/kuruma-log で公開
```

**メリット:**
- ✅ 完全無料（¥0）
- ✅ GitHub と統合
- ✅ 制限なし（個人利用）
- ✅ カスタムドメイン対応（無料）

**デメリット:**
- ❌ Vercel より遅い（デプロイ 2-3 分）
- ❌ サーバーレス関数未対応（純粋な静的ホスト）
- ❌ URL が「username.github.io/kuruma-log」（やや長い）

**費用:**
```
すべて無料（¥0）
```

**適用場面:**
```
✅ コスト重視
✅ GitHub を既に使っている
✅ カスタムドメインが不要
```

---

- ✅ カスタムドメイン対応

**デメリット:**
- ❌ 学習曲線（Vercel より若干複雑）
- ❌ 特に優位性がない

**費用:**
```
無料プラン: ¥0

```

**推奨度**: 🟡 中程度（Vercel がより簡単）

## 🚀 各選択肢でのセットアップ手順

### **Vercel の場合（最短）**

```bash
# 1. GitHub にリポジトリをアップロード

# 2. Vercel にサインイン（GitHub アカウント）
https://vercel.com

# 3. Import Project → GitHub リポジトリ選択

# 4. Deploy ボタンをクリック

# 完了（自動デプロイ開始）
→ 5分で https://kuruma-log.vercel.app で公開
```

---

### **GitHub Pages の場合**

```bash
# 1. GitHub リポジトリの Settings

# 2. Pages セクションで
   Source: GitHub Actions を選択

# 3. .github/workflows/deploy.yml を作成
```

---

### **VPS の場合**

```bash
# 1. VPS に SSH 接続
ssh -i ~/.ssh/key root@163.44.103.123

# 2. Node.js / npm をインストール
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 3. プロジェクトをクローン
git clone https://github.com/[username]/kuruma-log.git
cd kuruma-log
npm install

# 4. ビルド
npm run build

# 5. nginx 設定
sudo nano /etc/nginx/sites-available/kuruma-log
# dist/ をホストするよう設定

# 6. SSL 証明書取得（Certbot）
sudo certbot certonly --standalone -d kuruma-log.local

# 7. サービス起動
sudo systemctl restart nginx
```
