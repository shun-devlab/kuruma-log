# クルマログ 設計原則

**タイトル**: 「登録がめんどうにならない体験」を最優先  
**PM**: クロー  
**更新日**: 2026-03-01

---

## 🎯 核となる設計方針

### **最優先: 「体験」**

```
【階層】
1. 体験（記録時間、簡単さ） ← 最優先
2. 機能（次回予定、統計）   ← その次
3. デザイン（色、フォント）  ← その次
4. パフォーマンス             ← その次
```

**理由**: 「記録が簡単 = 毎日使う = 習慣化 = 価値提供」という好循環

---

## ⏱️ 記録時間の目標値（厳守）

### **目標 1: ホームウィジェット記録**

```
【理想】
アイコンをタップ → 完了（日付は自動で「今日」）
→ 3-5秒で終わる

【実装上の約束】
- 日付は「今日」で自動入力（変更可能）
- 走行距離は「前回から自動計算」で提案（洗車など不要な項目はスキップ）
- 金額は記録後に入力（ウィジェットではスキップ可）
```

---

### **目標 2: アプリ内記録**

```
【理想】
「新規記録」 → 種別選択 → 必須項目入力 → 保存
→ 20秒以内で終わる

【実装上の約束】
- 種別選択画面は「直感的」（迷わない）
- 必須項目は「メンテナンス種別ごと」に最小限化
- デフォルト値を活用（日付、走行距離など）
```

---

## 📋 メンテナンス種別ごとの必須・オプション定義

### **ガソリン給油**

**必須項目**:
- 日付（デフォルト: 今日）
- 走行距離（次回予定計算に必須。デフォルト: 前回 + 平均走行距離）

**オプション項目**:
- 金額（入力は「記録後の編集」で OK）
- メモ（給油地、使用した給油カードなど）

**記録時間目標**: 15-20秒

---

### **エンジンオイル交換**

**必須項目**:
- 日付（デフォルト: 今日）
- 走行距離（次回予定計算に必須。デフォルト: 前回 + 5000km）

**オプション項目**:
- 金額
- メモ（使用オイルの種類など）

**記録時間目標**: 15-20秒

---

### **タイヤ交換**

**必須項目**:
- 日付（デフォルト: 今日）

**オプション項目**:
- 走行距離（推奨ですが、覚えてなくても OK）
- 金額
- メモ（タイヤサイズ、ブランドなど）

**記録時間目標**: 10-15秒

---

### **洗車**

**必須項目**:
- 日付（デフォルト: 今日）

**オプション項目**:
- 走行距離（不要）
- 金額（手洗い自分で無料の場合は不要）
- メモ（外装、内装など）

**記録時間目標**: 5-10秒

---

### **故障・修理**

**必須項目**:
- 日付（デフォルト: 今日）

**オプション項目**:
- 走行距離
- 金額（後日記入でも OK）
- メモ（修理内容）

**記録時間目標**: 10-15秒

## 🧠 デフォルト値とスマート提案

### **パターン 1: 日付**

```
【実装】
すべてのメンテナンスで「日付」は自動で「今日」を入力
→ ユーザーは日付フィールドをスキップ可能
```

---

### **パターン 2: 走行距離（ガソリン・オイル）**

```
【実装】
前回のガソリン給油から「○○km」
→ 自動的に「前回走行距離 + ○○km」を提案
→ ユーザーは数字を確認して OK するだけ

例:
前回: 44,000km（1ヶ月前）
今回提案: 45,200km（今月走行距離の平均で計算）
→ ユーザーが実際の走行距離を確認
→ 違えば修正、同じなら OK
```

---

### **パターン 3: 金額（後日入力）**

```
【実装】
ガソリン給油やオイル交換の記録時に「金額はスキップできる」
→ 記録完了後、ウィジェット内または アプリ内で「金額を追加」
→ 後で思い出したら編集可能
```

## 🎨 UI/UX 設計の「体験重視」

### **原則 1: 選択式 > テキスト入力**

```
❌ ダメな例:
「何をしたか入力してください: [        ]」
→ キーボード開く、文字入力、時間かかる

✅ 良い例:
「何をしましたか?」
[⛽ ガソリン] [🛢️ オイル] [🛞 タイヤ] [🚿 洗車] [⚠️ 修理]
→ タップするだけ、即座に選択完了
```

---

### **原則 2: プログレッシブディスクロージャー**

```
【段階的に情報を提示】

ステップ 1: メンテナンス種別を選ぶ
ステップ 2: 必須項目を入力
ステップ 3: 「詳細を追加」で オプション項目（後で編集可）

→ 最初は「最小限の入力」で完了させる
```

---

### **原則 3: 確認より「修正」を優先**

```
【パターン A（悪い）】
入力フォーム → [確認] ボタン → 確認画面 → [保存]
→ 4 ステップ、時間かかる

【パターン B（良い）】
入力フォーム → [保存] → 完了画面 → [詳細を編集] 
→ 基本は 2 ステップ、後で修正可能
```

## 📊 ホームウィジェット：3-5秒の秘密

### **工夫 1: アイコンをタップするだけ**

```
【ウィジェット画面】
┌───────────────────┐
│ ⛽  🛢️  🛞  🚿  ⚠️ │
└───────────────────┘
→ 1 タップで「記録」開始
→ 日付は自動で「今日」
```

---

### **工夫 2: シンプルメンテナンスは「即記録」**

```
【洗車の場合】
① ウィジェットの 🚿 をタップ
② 確認画面: 「2026-03-01 洗車を記録します？」
③ [はい] をタップ
④ 完了！
→ 5秒以内

【ガソリンの場合】
① ウィジェットの ⛽ をタップ
② 走行距離入力: 「45,200km」（前回から自動計算）
③ [保存] をタップ
④ 完了
→ 3-5秒（走行距離が正しければ修正なし）
```

---

### **工夫 3: アプリ内での「詳細追加」は後で OK**

```
ウィジェットで「ガソリン給油: 45,200km」を記録
    ↓
後で（今夜、明日でも OK）アプリを開く
    ↓
「詳細を編集」で金額やメモを追加
    ↓
完了

→ 最初の記録は「本当に最小限」、詳細は後から
```

## 💡 「体験重視」がもたらす価値

```
【ユーザー行動の変化】
記録が簡単
  ↓
毎日使うようになる（習慣化）
  ↓
「あ、次のオイル交換まで 1,200km だ」と気づく
  ↓
メンテナンスを計画的に実施
  ↓
車が長持ちする＆家計が見える

【ビジネス的な価値】
"簡単さ" = 競争優位性
Drivvo ユーザーの乗り換え
口コミで「シンプルで速い」と拡散
```
