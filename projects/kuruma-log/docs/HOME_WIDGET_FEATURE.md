# クルマログ ホームウィジェット機能

**機能名**: ホームスクリーン ウィジェット（クイック記録）  
**導入時期**: Phase 1（MVP に含める）  
**差別化度**: ⭐⭐⭐⭐⭐ 非常に高い  
**PM**: クロー

---

## 🎯 機能コンセプト

```
【現在（Drivvo）】
1. アプリを開く
2. [新規記録] をタップ
3. メンテナンス種別を選ぶ
4. 必須項目を入力
5. [保存] をタップ
→ 約 30-60 秒かかる

【クルマログ（ウィジェット）】
1. ホーム画面のウィジェットを見る
2. [⛽] または [🚿] をタップ
3. 完了！（日付は自動で今日）
→ 約 3-5 秒で完了
```

**価値**: 「思いついたら即記録」ができる

## 🔧 実装仕様

### **iOS ウィジェット（WidgetKit）**

**技術**:
- WidgetKit（iOS 14+）
- SwiftUI
- App Groups で データ共有

**実装ステップ**:
```
1. Widget Extension を作成
2. TimelineEntry でウィジェット更新
3. Intent Configuration で カスタマイズ可能に
4. App Groups で アプリと データ共有
5. Deep Link で アプリ起動時に詳細画面へ
```

---

### **Android ウィジェット**

**技術**:
- AppWidget Framework
- RemoteViews
- PendingIntent

**実装ステップ**:
```
1. AppWidgetProvider を実装
2. レイアウト XML を定義
3. BroadcastReceiver で クリック処理
4. Shared Preferences で データ共有
5. App Shortcuts で クイック起動
```


## 🎯 意思決定事項

### **確認事項 1: MVP にウィジェット機能を含める？**

案:
- ✅ A: 含める（差別化が大きい。27h の追加工数）
- 🟡 B: Phase 2 に延期（今は基本機能だけ）
- ❌ C: 不要（複雑すぎる）

→ **推奨: A（含める）** — Drivvo との圧倒的な差別化

---

### **確認事項 2: 最初は「iOS のみ」で、Android は後？**

案:
- ✅ A: iOS 優先（iPhone ユーザーが多い）→ Android は Phase 2
- 🟡 B: 同時実装（工数 +27h）
- ❌ C: Android 優先

→ **推奨: A** — スケジュール現実的

---

### **確認事項 3: ウィジェットで「即記録」か「詳細入力」か？**

案:
- ✅ A: 洗車などシンプルな項目は「即記録」、ガソリン・オイルは「詳細入力」
- 🟡 B: 全部「詳細入力」（ウィジェットの意味が薄れる）
- 🟡 C: 全部「即記録」（必須項目が入力されない）

→ **推奨: A** — メンテナンス種別ごとに最適化
