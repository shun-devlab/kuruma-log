# クルマログ UI フロー詳細設計

**設計方針**: ぴよログの「タイムライン + 記録ボタン一画面完結」  
**最優先**: 「開いたらすぐ記録できる」体験  
**PM**: クロー

---

## 🎯 UI/UX の核となるコンセプト

```
【ぴよログから学んだ設計】
- ホーム画面 = 全機能の中心
- タイムライン（記録履歴）+ 記録ボタン（新規記録）が同一画面
- 「どこかのボタンを押す」必要なし
- 開いたらすぐ記録できる
- スクロールで過去の記録も見える

【クルマログの実装】
1. ホーム画面を開く
2. 次回予定を確認（目立つバナー）
3. タイムラインで過去記録を確認
4. 「⛽」「🛢️」などのアイコンをタップして即記録
→ すべて 1 画面内で完結
```

---

## 📱 ホーム画面の構成（詳細）

### **1. ヘッダー（固定）**

```
┌──────────────────────────┐
│ 🚗 クルマログ             │← アプリタイトル
│ 2026-03-01 (日)  [⚙️]    │← 本日日付 + 設定ボタン
└──────────────────────────┘
```

---

### **2. 次回予定バナー（視覚的に目立つ）**

```
┌──────────────────────────┐
│ 【次回予定】⚠️           │← 黄色または赤色背景
│                          │
│ 🛢️ オイル交換           │← 最も近い予定を先頭
│    あと 1,200km          │
│    2026-04-15 予想       │
│                          │
│ 🛞 タイヤ交換           │← 次の予定
│    あと 12,000km         │
│    2026-06-01 予想       │
└──────────────────────────┘
```

**デザイン:**
- 背景色: 淡いオレンジ / イエロー
- テキスト: ダークグレー
- アイコン: 大きく目立つ
- **クリック可能**: タップすると設定画面へ（後で修正可能）

├──────────────────────────┤

│  └ 金額: ¥3,500         │

│                          │
│ 2026-02-28               │
│ 🚿 洗車                  │
│  └ [編集] [削除]        │
│                          │
│ 2026-02-20               │
│ ⚠️ 修理                  │
│  └ 内容: エンジン異音    │
│  └ 金額: ¥15,000        │
│  └ [編集] [削除]        │
│                          │
│ 2026-02-10               │
│ 🛢️ オイル交換           │
│  └ 走行距離: 40,200km   │

**特徴:**
- 新しい順に表示（上が最新）

- スクロールで過去の記録も確認可能

### **5. 記録ボタンエリア（常に下部に固定）**

```
┌──────────────────────────┐

│ガソリン オイル タイヤ 洗車 修理

**特徴:**
- 画面下部に常に表示（スクロールで追従）

### **6. ボトムナビゲーション（固定）**

```
┌──────────────────────────┐
│ [ホーム] [一覧] [統計] [設定] │

**特徴:**

- 統計: グラフ・分析

## ⚡ インタラクション詳細

### **パターン A: ホーム画面から記録（⛽ ガソリン給油）**

```
【ステップ 1】ホーム画面を開く
┌──────────────────────────┐
│ 🚗 クルマログ             │

│ ⛽ 🛢️ 🛞 🚿 ⚠️      │
└──────────────────────────┘

【ステップ 2】⛽ をタップ
    ↓
【記録パネルが展開（インライン）】
┌──────────────────────────┐

├──────────────────────────┤

│ 走行距離: [45,200] km    │← 前回から自動提案
│ 金額: [3,500] 円        │← オプション
│ メモ: [         ]       │← オプション
│                          │
│ [保存] [キャンセル]      │

    ↓
「45,200km でいい」なら、走行距離は変更なし

【ステップ 4】[保存] をタップ
    ↓
【ホーム画面に戻る】
記録がタイムラインの最上位に追加される
ぴよログのように自動で上にスクロール

合計時間: 3-5秒 ✅
```

## 📊 ホーム画面のスクロール挙動

```
【初期状態】
┌──────────────────────────┐
│ ヘッダー（固定）         │
├──────────────────────────┤
│ 次回予定バナー           │← 常に見える
├──────────────────────────┤
│ タイムライン（スクロール可）
│ ...                      │
└──────────────────────────┘

【スクロール時】
ユーザーがタイムラインを上にスクロール
  ↓
「次回予定」バナーは上に収納（sticky）
記録ボタンエリアは下に固定（常に見える）

【新規記録後】
自動でトップにスクロール
新しい記録がタイムラインの最上位に表示
```

## 🐾 実装ポイント

### **フロント実装での注意点**

1. **記録パネルのモーダル化**
   - `<transition>` で スライドアップアニメーション
   - 背景は `z-index` で制御
   - ESC キーで閉じる

2. **スクロール位置の保持**
   - 記録後は自動トップへスクロール
   - ぴよログのような「新しい記録の自動表示」

3. **フォーム の自動入力**
   - 日付: `new Date().toISOString()`
   - 走行距離: API で前回値を取得 → +自動計算値

4. **レスポンシブデザイン**
   - モバイル（375px）: 縦一列
   - タブレット（768px+）: 2 列レイアウト（オプション）
