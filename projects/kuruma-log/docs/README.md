### 🎨 **UI/UX 設計ドキュメント**

#### 8. **UI_DESIGN_REFERENCE.md** (12 KB) - ぴよログから学ぶ UI 設計
**内容:**
- **ぴよログの 4 つのシンプル戦略**:
  1. 事前に選択肢を用意（テキスト入力より選択式）
  2. タイムライン形式（時系列で見える）
  3. 下部タブ 4-5 個のみ
  4. クイック記録ボタン
- **クルマログの UI/UX 設計方針**:
  - アイコン + テキスト
  - 選択式 > テキスト入力
  - プログレッシブディスクロージャー
- **ホーム画面レイアウト案**:
  ```
  【次回予定】（オレンジで目立たせる）
  【最近の記録】（タイムライン）
  【月別支出】
  [➕ 新規記録]
  [ホーム][一覧][統計][設定]
  ```
- **カラースキーム**: 淡いカーブルー + グリーン・オレンジアクセント

**ステータス**: ✅ 完成

#### 9. **HOME_WIDGET_FEATURE.md** (11 KB) - ホームウィジェット機能
**内容:**
- **ウィジェット機能の概要**:
  - ホーム画面から 3-5 秒で記録完了
  - アイコンをタップするだけ
  - 日付は自動で「今日」
- **iOS ウィジェット**（WidgetKit）:
  - 小（2x2）・中（2x4）・大（4x4）
- **Android ウィジェット**:
  - 小（2x2）・大（4x3）
- **実装仕様**:
  - ガソリン・オイル: 詳細入力（走行距離必須）
  - 洗車・修理: 即記録（日付のみ）
- **MVP への包含**:
  - ✅ Android（Phase 1）
  - ⏳ iPhone（Phase 2）
- 工数: +27h（MVP に追加）

**ステータス**: ✅ 完成

#### 10. **DESIGN_PRINCIPLES.md** (7.5 KB) - 設計原則
**内容:**
- **最優先設計方針**: 「登録がめんどうにならない体験」
- **記録時間の目標値**（厳守）:
  - ホームウィジェット: 3-5 秒
  - アプリ内記録: 20 秒以内
- **メンテナンス種別ごとの必須・オプション定義**:
  - ガソリン: 日付+走行距離（必須）
  - オイル: 日付+走行距離（必須）
  - タイヤ: 日付（必須）
  - 洗車: 日付（必須）
  - 修理: 日付（必須）
- **スマート提案**（デフォルト値の活用）:
  - 日付: 自動で「今日」
  - 走行距離: 「前回 + 平均」で提案
  - 金額: スキップ可能
- **UI/UX の 5 つのルール**:
  1. 選択式 > テキスト入力
  2. プログレッシブディスクロージャー
  3. 確認より「修正」を優先
  4. アイコン + テキスト
  5. 下部タブ 4 個以下

**ステータス**: ✅ 完成

## 🐾 クローのメモ

### **判断軸（常に思い出す）**

```
「これはユーザーの記録体験をシンプルで、
  手軽に、速くしているか？」
  
Yes → 実装
No  → 再検討
```

### **クルマログの最大の武器**

1. **ウィジェット**（3-5秒記録） → Drivvo に絶対ない
2. **シンプル UI**（ぴよログ級） → Drivvo より直感的
3. **完全日本語** → ネイティブ対応
4. **走行距離オプション化** → 記録の敷居を低く
